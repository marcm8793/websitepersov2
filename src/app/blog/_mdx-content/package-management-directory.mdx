## Dependency Resolution Libraries

Reusable libraries that solve the version constraint satisfaction problem.

| Library                                                               | Description                                     | Used By                              |
| --------------------------------------------------------------------- | ----------------------------------------------- | ------------------------------------ |
| [PubGrub](https://github.com/dart-lang/pub/blob/master/doc/solver.md) | Conflict-driven solver with good error messages | Dart's pub, Poetry, uv, Hex, Bundler |
| [libsolv](https://github.com/openSUSE/libsolv)                        | SAT-based solver                                | DNF, Zypper, Conda, Mamba            |
| [Rattler](https://github.com/mamba-org/rattler)                       | Rust implementation of Conda package management | Pixi                                 |
| [Molinillo](https://github.com/CocoaPods/Molinillo)                   | Backtracking resolver tuned for Ruby            | Older Bundler, CocoaPods             |
| [Clingo](https://potassco.org/clingo/)                                | Answer set programming solver                   | Spack                                |
| [Resolvelib](https://github.com/sarugaku/resolvelib)                  | pip's backtracking resolver                     | pip (since 20.3)                     |
| [CUDF](https://www.mancoosi.org/cudf/)                                | Common Upgradeability Description Format        | opam with external solvers           |
| [resolvo](https://github.com/mamba-org/resolvo)                       | SAT solver for package management               | Mamba team projects                  |

## Trusted Publishing

Infrastructure for verifying package provenance and integrity.

### Sigstore

[Keyless signing infrastructure](https://www.sigstore.dev/) including [cosign](https://github.com/sigstore/cosign), [fulcio](https://github.com/sigstore/fulcio), and [rekor](https://github.com/sigstore/rekor). Used by npm, PyPI, and others for provenance. [policy-controller](https://docs.sigstore.dev/policy-controller/overview/) enforces signature policies in Kubernetes.

### The Update Framework (TUF)

[Framework for secure software update systems](https://theupdateframework.io/). Used by PyPI, RubyGems, Homebrew.

### in-toto

[Supply chain layout and verification](https://in-toto.io/). Ensures each step in the build pipeline was performed correctly.

### SBOMit

[Generates signed, in-toto attested SBOMs](https://github.com/interlynk-io/sbomit).

### Ecosystem-Specific

| Ecosystem | Trusted Publishing                                                                                                        |
| --------- | ------------------------------------------------------------------------------------------------------------------------- |
| **Go**    | [sum.golang.org](https://sum.golang.org/) - Transparency log for module checksums                                         |
| **npm**   | [npm provenance](https://docs.npmjs.com/generating-provenance-statements) - Links packages to source commits via Sigstore |
| **PyPI**  | [Trusted Publishers](https://docs.pypi.org/trusted-publishers/) - OIDC-based publishing from CI providers                 |
